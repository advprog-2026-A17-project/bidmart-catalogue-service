# API — Modul 2 (Katalog)

Base path: `/api/v1/catalogue`

OpenAPI: [openapi.yaml](./openapi.yaml)

**Listing = identitas lelang di katalog.** Data produk, harga tampilan, reserve, waktu, dan lifecycle (`DRAFT` → `ACTIVE` → `EXTENDED` → `CLOSED` → `WON` / `UNSOLD` / `CANCELLED`). Modul 3 memvalidasi status dan menerima update harga via event.

## Listings — `/api/v1/catalogue/listings`

| Method | Path | Keterangan |
|--------|------|------------|
| POST | `/` | Buat listing |
| GET | `/` | Daftar semua |
| GET | `/{id}` | Detail |
| GET | `/{id}/summary` | Ringkasan untuk validasi lelang |
| GET | `/search` | keyword, category, price, status, `endingWithinHours` |
| PUT | `/{id}` | Update (terbatas jika ada bid) |
| DELETE | `/{id}` | Hapus |
| POST | `/{id}/cancel` | Batalkan (tanpa bid aktif) |
| POST | `/{id}/publish` | DRAFT → ACTIVE |
| POST | `/{id}/deactivate` | ACTIVE → DRAFT |
| POST | `/{id}/extend` | Anti-sniping → EXTENDED |
| POST | `/{id}/close` | → CLOSED |
| POST | `/{id}/won` | Body `finalPrice` |
| POST | `/{id}/unsold` | → UNSOLD |
| POST | `/{id}/bid` | Update harga (internal/event) |

## Kategori — `/api/v1/catalogue/categories`

## gRPC

Port `9091` — ringkasan listing untuk modul 3/4 (internal).

## Event masuk (RabbitMQ)

- `auction.bid-placed.v1` — update `currentPrice`, `hasBids`
- `auction.ended.v1` — status WON/UNSOLD
