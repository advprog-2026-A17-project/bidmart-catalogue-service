# Software Design

## Layering

- **Controllers:** `controller/ListingController.java`, `CategoryController.java`
- **Services:** `service/ListingServiceImpl.java`, `CategoryServiceImpl.java` implementing `ListingService`, `CategoryService`
- **Domain:** `model/Listing.java`, `model/Category.java`
- **Persistence:** `repository/ListingRepository.java` (+ `JpaSpecificationExecutor`), `CategoryRepository.java`
- **Integration:** `event/AuctionCatalogueEventConsumer.java`, `grpc/CatalogueGrpcService.java`

## SOLID

- **SRP:** `CatalogAccessPolicy` centralizes seller/buyer visibility rules; controllers do not embed authorization logic.
- **OCP:** New listing filters added via `ListingSpecification` without changing repository interfaces.
- **LSP:** `ListingServiceImpl` substitutable wherever `ListingService` is injected.
- **ISP:** gRPC service exposes only summary operations required by auction (`CatalogueGrpcService`).
- **DIP:** Controllers depend on `ListingService` interface, not `ListingServiceImpl`.

## Design patterns (≥3)

### 1. Repository + Specification

- `ListingRepository` + `specification/ListingSpecification.java` for dynamic queries (category, status, seller).

### 2. Strategy (auth resolution)

- `config/AuthInterceptor.java` — Strategy 1: gateway `X-User-Id`; Strategy 2: direct JWT parse fallback.

### 3. Observer / application events

- `event/ListingEventPublisher.java` wraps Spring `ApplicationEventPublisher` for `ListingCreatedEvent`.

### 4. Event consumer (async integration)

- `event/AuctionCatalogueEventConsumer.java` updates listing status when auction ends (Rabbit listener).

### 5. gRPC adapter

- `grpc/CatalogueGrpcService.java` — adapter from domain `ListingService` to protobuf RPC for auction service.
