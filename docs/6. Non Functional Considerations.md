# Non Functional Considerations

## Performance

- Listing search uses JPA Specification — index `seller_id`, `status` in production DB tuning.
- gRPC path avoids JSON serialization for auction hot reads.

## Security

- All mutating routes behind `AuthInterceptor`.
- Public read endpoints still respect `CatalogAccessPolicy` for draft/hidden listings.

## Reliability

- Event consumer idempotency: status updates should tolerate duplicate `AuctionEnded` deliveries.
- DB migrations via `SPRING_JPA_HIBERNATE_DDL_AUTO` in local compose (`update`); production should use explicit migrations.

## Scalability

- Stateless HTTP/gRPC instances behind gateway.
- PostgreSQL connection pool per instance (Spring Boot defaults).
